# Support

## Bugs

Open a bug report with:

- Extension version.
- VS Code version.
- Operating system.
- `haproxy.version` setting.
- A minimal sanitized HAProxy config snippet.
- Expected behavior and actual behavior.

Use the bug template: <https://github.com/JuanTorchia/gmm-haproxy-vscode/issues/new/choose>

## Feature Requests

Open a feature request when the change would reduce mistakes while editing HAProxy configs. Good requests explain the production risk, workflow, or HAProxy version difference involved.

## Directive Data Updates

If a directive, option, or version rule is missing, use the directive data template. Link to official HAProxy documentation whenever possible.

## Security Issues

Do not open public issues for vulnerabilities. See [SECURITY.md](SECURITY.md).
